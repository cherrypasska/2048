# Описание правил игры 2048
Игра "2048" - это популярная головоломка, в которой игроку нужно
объединять плитки с одинаковыми числами, чтобы достигнуть плитки с
числом 2048. Вот полные правила игры:

# Начало игры:
На игровом поле размером 4x4 появляются две плитки со значением
"2" или "4" (шанс появления 2 – 90%).

Цель игры - объединять плитки с одинаковыми числами, чтобы создавать
плитки с бОльшими числами и достигнуть плитки с числом "2048".

# Движение:
Игрок может сдвигать все плитки на игровом поле в одном из четырех
направлений: вверх, вниз, влево или вправо.

При каждом ходе все плитки двигаются на максимальное расстояние в
выбранном направлении, пока не встретятся с другой плиткой или краем
игрового поля и появляется одна новая плитка(90% шанс 2, 10% шанс 4).

# Слияние плиток:
Если две плитки с одинаковыми числами соприкасаются в результате
движения, они объединяются в одну плитку с суммой их значений.

Например, если две плитки с числами "2" и "2" соприкасаются, они
объединяются в одну плитку с числом "4".

# Очки:
Игрок получает очки за каждое успешное слияние плиток.

Чем выше число на объединенной плитке, тем больше очков получает игрок.

# Проигрыш:
- Игрок проигрывает, если все ячейки игрового поля заполняются и
больше нет возможности для дальнейших ходов.

- Цель игры - достичь плитки с числом "2048", но игра продолжается
после достижения этой цели.

# Победа:
- Игрок выигрывает, если достигает плитки с числом "2048". Однако
после этого игра продолжается.

Игра "2048" требует логического мышления и стратегического планирования,
чтобы достичь высоких результатов.

# Технологии:
Использование HTML для разметки игрового поля и интерфейсных
элементов. 

Использование CSS для стилизации игры и создания анимаций.

Использование JavaScript для логики игры и обработки
пользовательских вводов.

Адаптивность и кроссбраузерность:

Игра должна быть адаптивной и корректно отображаться на различных
устройствах (компьютеры, планшеты, смартфоны).
Игра должна поддерживать основные веб-браузеры, такие как Chrome,
Yandex и Edge.

Анимации:

Добавить анимации для плавного появления и исчезновения плиток
при их появлении и объединении.

Описание классов, идентификаторов, селекторов атрибутов и ссылок:
Классы:

- score-container: Контейнер для отображения счета игры.

- Game-board: Область игрового поля.

- grid: Контейнер с сеткой игровых плиток.

- cap: Класс для игровых плиток. Каждая плитка имеет атрибуты данных
(`data-rows` и `data-columns`) для указания их позиции в сетке.

- game-over: Область, отображаемая при проигрыше.
Идентификаторы:

- cur_cont: Идентификатор для текущего счета.

- current-score: Идентификатор для отображения текущего счета внутри
контейнера `cur_cont`.

- high_cont: Идентификатор для лучшего счета.

- high-score: Идентификатор для отображения лучшего счета внутри
контейнера `high_cont`.

- restart: Идентификатор для кнопки рестарта.

- game-over: Идентификатор для области, отображаемой при проигрыше.

- restart-btn: Идентификатор для кнопки перезапуска в области проигрыша.
Селекторы:

- body: Селектор для тега `<body>`.

- h1: Селектор для заголовка первого уровня.

- .score-container div: Селектор для всех дочерних элементов `<div>` внутри
контейнера с классом `score-container`.

- .Game-board .grid .cap: Селектор для всех элементов с классом `cap`
внутри контейнера `.grid`, который, в свою очередь, находится в `.Game-
board`.

- #game-over button: Селектор для кнопки перезапуска в области
проигрыша.

Атрибуты:

- data-rows: Атрибут данных для указания строки, в которой расположена
игровая плитка.

- data-columns: Атрибут данных для указания столбца, в котором
расположена игровая плитка.

Ссылки:

- main.css: Ссылка на внешний файл стилей.

- scripts.js: Ссылка на внешний файл скриптов.
